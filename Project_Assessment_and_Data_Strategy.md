# Django 电影推荐系统项目评估与数据策略

## 1. 项目评估结论

### 可行性分析
- **成功率**：极高。基于 Python Django + 经典推荐算法（UserCF/ItemCF）的技术栈非常成熟，AI 协作开发效率极高。
- **开发周期预估**：
  - **全职投入**：约 2 周完成 MVP（最小可行性产品）。
  - **兼职/课余**：约 3-4 周。
- **技术难点**：核心在于协同过滤算法的实现与数据冷启动问题的处理。

### 权限与限制
- **爬虫限制**：豆瓣/IMDb 等网站有严格反爬机制，不建议作为核心数据源，仅作为补充。
- **API 限制**：若使用 TMDB/OMDB API 补全海报，需申请免费 Key（通常够用）。
- **本地环境**：无特殊限制，支持 Windows/Mac/Linux。

---

## 2. 核心数据需求

为了实现协同过滤推荐，系统必须包含以下三类数据表：

### A. 电影元数据表 (Movies)
用于前端展示，必须包含。
- `Movie_ID`: 唯一标识
- `Title`: 电影名称
- `Genres`: 类别（动作/喜剧等）
- `Poster_URL`: 海报图片链接（需补全）
- `Overview`: 剧情简介（需补全）

### B. 用户-评分矩阵表 (Ratings)
**算法核心燃料**，没有此数据无法计算相似度。
- `User_ID`: 用户标识
- `Movie_ID`: 电影标识
- `Rating`: 评分（1-5分）
- `Timestamp`: 时间戳

### C. 用户信息表 (Users)
用于注册登录及基础画像。
- `User_ID`: 唯一标识
- `Username`: 用户名
- `Password`: 密码（加密存储）

---

## 3. 数据获取策略（混合方案）

建议采用 **“MovieLens 数据集 + API/爬虫补全”** 的混合策略。

### Step 1: 核心数据 (MovieLens)
直接下载学术界标准的 **MovieLens 100K** 或 **ml-latest-small** 数据集。
- **获取内容**：包含完整的 `User_ID`, `Movie_ID`, `Rating` 关系数据，直接解决算法冷启动问题。
- **下载地址**：[MovieLens Datasets](https://grouplens.org/datasets/movielens/)

### Step 2: 视觉数据补全 (API/爬虫)
MovieLens 只有文本数据，缺乏海报和简介。
- **方案 A (推荐)**：使用 TMDB API。编写脚本读取 MovieLens 的电影名，调用 API 获取高清海报 URL。
- **方案 B (备选)**：编写轻量级爬虫，根据电影名去豆瓣/IMDb 抓取图片链接。

---

## 4. 推荐执行路线

1.  **环境搭建**：初始化 Django 项目，配置 MySQL/SQLite 数据库。
2.  **数据入库**：编写脚本解析 MovieLens 数据，导入数据库。
3.  **算法实现**：编写 Python 脚本实现 UserCF 推荐逻辑（输入用户ID -> 输出推荐电影ID列表）。
4.  **Web 开发**：开发前端页面，调用算法接口展示推荐结果。
5.  **数据补全**：最后通过 API 批量更新电影海报，提升界面美观度。
