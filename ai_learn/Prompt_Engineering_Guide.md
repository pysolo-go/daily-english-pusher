# 大模型基础与 Prompt Engineering (提示工程) 实战指南

## 1. 核心概念：大模型的“脾气”

在写提示词之前，你必须理解大模型（LLM）是如何工作的。它不是搜索引擎，它是一个**“概率预测机”**。

### 1.1 Token (词元) —— 计费与计算的单位
*   **定义**：大模型不读字，读 Token。英文单词通常是 1 个 Token，汉字通常是 1-2 个 Token。
*   **影响**：
    *   **计费**：你问的越多，它答的越多，花的钱越多。
    *   **上下文窗口 (Context Window)**：模型记性是有限的（如 4k, 128k, 1m）。超过这个长度，前面的对话它就忘了。

### 1.2 Temperature (温度) —— 理性 vs 感性
*   **范围**：通常是 0.0 到 1.0 (有些模型到 2.0)。
*   **低温度 (0.1 - 0.3)**：**严谨、确定**。适合写代码、数学题、数据提取。
    *   *AI 表现*：每次回答几乎一样。
*   **高温度 (0.7 - 1.0)**：**发散、创意**。适合写小说、头脑风暴、聊天。
    *   *AI 表现*：每次回答都不一样，容易产生“幻觉”。

## 2. 提示工程进阶技巧 (从入门到精通)

### 2.1 Zero-shot (零样本) vs Few-shot (少样本/ICL)
这是提升效果最快的方法。**给例子 (In-Context Learning)**。

*   **❌ Zero-shot (直接问)**:
    > "把这句话翻译成英文：'今天天气不错'。"
*   **✅ Few-shot (给几个例子)**:
    > "请模仿下面的风格进行翻译：
    > 中文：你好吗？ -> 英文：How do you do?
    > 中文：吃了吗？ -> 英文：Have you eaten yet?
    > 中文：今天天气不错 -> 英文："

### 2.2 Chain of Thought (CoT, 思维链)
强迫模型展示思考过程，能显著提高逻辑题准确率。

*   **核心咒语**：**"Let's think step by step" (让我们一步步思考)**。
*   **示例**:
    > "我去市场买了10个苹果，吃了2个，又送给邻居3个，然后我又去买了5个。我现在有几个苹果？
    > **请一步步思考并计算。**"

### 2.3 结构化提示词 (Structured Prompt)
像写代码一样写提示词。这是目前最专业的写法（LangGPT 范式）。

```markdown
# Role (角色)
你是一位资深的 Python 面试官。

# Profile (画像)
- 作者: Solo
- 版本: 1.0
- 语言: 中文
- 描述: 专注于考察候选人的基础知识与工程思维。

# Rules (规则)
1. 每次只问一个问题。
2. 等候候选人回答后，再进行点评和追问。
3. 保持语气专业但友善。

# Workflow (工作流)
1. 首先让候选人做自我介绍。
2. 根据自我介绍，从 Python 基础、并发编程、数据库中选择一个方向提问。
3. 对回答打分（0-10分）。

# Initialization (初始化)
我是面试官。请候选人开始介绍自己。
```

## 3. 实战演练：坏 Prompt vs 好 Prompt

### 场景：写一个贪吃蛇游戏

*   **❌ 坏 Prompt (小白)**:
    > "帮我写个贪吃蛇游戏。"
    *   *结果*：AI 可能会给你一段不完整的代码，或者用你不喜欢的库（比如 C++）。

*   **✅ 好 Prompt (工程师)**:
    > "你是一位资深的 Python 游戏开发者。
    > 请使用 `pygame` 库编写一个完整的贪吃蛇游戏。
    >
    > **要求**：
    > 1. 窗口大小 800x600。
    > 2. 蛇的颜色是绿色，食物是红色。
    > 3. 包含计分系统，显示在左上角。
    > 4. 游戏结束后按 'R' 键可以重开。
    > 5. 代码需要有详细的中文注释。
    >
    > 请一步步思考，先列出代码结构，然后给出完整可运行的代码。"
